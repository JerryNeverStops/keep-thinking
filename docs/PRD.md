# 哲学原书批注站 PRD

## 1. 背景

这个项目是一个个人哲学读书笔记站。它不是传统博客，也不是文章时间线，而是以可公开展示的哲学原书为中心，把原文、划线、批注、概念索引和搜索组织成一个公开阅读空间。

第一版采用 GitHub Pages 静态发布：维护者在本地阅读和批注，通过 GitHub 仓库更新触发网站构建；读者只访问公开站点，不登录、不评论、不创建批注。

## 2. 目标

- 让维护者在本地获得优秀的阅读和批注体验。
- 让读者在网页上同时看到原书和整理后的批注。
- 让每本书形成书籍总览、阅读工作区、概念索引和全文搜索入口。
- 让笔记可以派生为微信公众号文章草稿，但不改变主站的原书中心结构。

## 3. 非目标

- 第一版不做网页端管理员后台。
- 第一版不做在线上传、在线保存或数据库持久化。
- 第一版不做读者评论、点赞、收藏或多人协作。
- 第一版不把公众号文章作为主站文章栏目公开。
- 第一版不支持任意版权书籍公开展示，只发布公版、开放授权或已获授权文本。

## 4. 角色

**维护者**：仓库所有者，负责在 Zotero 中阅读、划线、写批注，运行导出命令，提交 GitHub 更新。

**读者**：公开访问者，只能阅读已经发布的原书、批注、概念索引和搜索结果。

## 5. 核心工作流

1. 维护者在 Zotero 中阅读 PDF 或 EPUB。
2. 维护者使用固定颜色进行划线和批注。
3. 维护者运行手动导出命令，把 Zotero 批注转换成仓库中的书籍包数据。
4. 维护者本地预览 VitePress 站点。
5. 维护者提交并推送到 GitHub。
6. GitHub Pages 构建并公开更新后的站点。
7. 维护者可以从一段原文和一组笔记生成微信公众号文章草稿，之后人工润色并发布到公众号。

## 6. 内容模型

每本书是一个 **Book Package**，建议结构：

```text
books/
  kant-critique-pure-reason/
    source.pdf
    book.yaml
    notes.generated.md
    search.generated.json
```

`book.yaml` 维护发布相关信息：

```yaml
zoteroKey: ABCD1234
slug: kant-critique-pure-reason
title: 纯粹理性批判
author: 伊曼努尔·康德
translator: ""
language: zh
sourceFile: source.pdf
format: pdf
publicationEligibility:
  type: public-domain
  note: ""
topicTags: [认识论, 康德]
readingStatus: reading
summary: ""
```

Zotero item key 用于机器关联，slug 用于公开 URL。书名和作者不作为唯一匹配依据。

## 7. 笔记类型和颜色

Zotero 颜色映射为网站笔记类型：

- 黄色：解释
- 蓝色：问题
- 紫色：概念
- 绿色：论证
- 粉色：联想
- 灰色或无笔记高亮：独立划线 / 待整理

笔记类型完全由颜色判断；批注正文保持自然语言。可选 hashtag 会被导出为标签。

紫色 Concept Note 的第一行作为概念名，并进入书内概念索引。

没有正文的独立高亮只显示在原文中，不进入右侧笔记流。

## 8. 网站信息架构

首页是书架，不是文章时间线。

书架按阅读状态分组，再按最近批注或导出时间排序：

1. 正在读
2. 暂停 / 搁置
3. 已读完

每本书至少有：

- 书籍总览页：元数据、简介、阅读状态、笔记数量、概念索引、筛选入口。
- 阅读工作区：左侧原文，右侧笔记流。

主站界面使用中文；书籍原文语言作为元数据保留。

## 9. 阅读工作区

桌面端：

- PDF 显示原始固定版式阅读器。
- EPUB 显示可重排网页阅读器。
- 右侧显示有正文的阅读笔记流。
- 点击右侧笔记，左侧定位到对应原文并提示高亮。
- 点击左侧高亮，右侧定位到对应笔记。
- 左右不强制同步滚动。

移动端：

- 原文为主阅读区域。
- 笔记以弹窗形式出现。
- 保留从原文到笔记、从笔记回原文的定位能力。

EPUB 第一版支持基础字号调节和明暗主题，不做完整电子书排版设置。

## 10. 搜索

第一版做静态全文搜索索引，在浏览器本地搜索，不依赖后端。

搜索范围：

- 书籍元数据
- 阅读笔记
- 可抽取的 PDF / EPUB 原文文本

结果排序采用笔记优先：

1. 命中的阅读笔记，尤其是概念和论证笔记
2. 命中的原文片段
3. 命中的书籍元数据

搜索结果只展示短片段，并能跳回对应书和阅读位置。

## 11. 微信公众号文章草稿

公众号文章是从原书批注派生出的 **Article Draft**，不是主站核心内容。

文章草稿来源于：

- 一大段原文上下文
- 一组相关阅读笔记

文章风格：

- 学术论文式
- 概念准确
- 论证严谨
- 中等比例引用原文
- 使用轻量脚注
- 文末列出依据文本

文章草稿放在仓库的 `articles/` 目录，但第一版不在网站公开导航中展示。

建议结构：

```text
articles/
  2026-06-kant-synthetic-a-priori.md
```

建议 frontmatter：

```yaml
sourceBook: kant-critique-pure-reason
sourcePassage: ""
noteIds: []
channel: wechat
status: draft
style: academic
```

生成方式：本地命令生成草稿，人工润色后发布到公众号。

## 12. 技术路线

- VitePress 作为静态站框架。
- 自定义阅读组件负责 PDF / EPUB 阅读页。
- GitHub Pages 负责公开托管。
- Zotero 负责本地阅读和批注。
- 手动导出命令负责从 Zotero 生成站点数据。
- 仓库保存可公开的 PDF / EPUB 原文文件。

## 13. 第一版验收标准

- 可以在本地启动 VitePress 站点。
- 首页展示书架，并按阅读状态分组。
- 至少一本文本可以展示书籍总览页。
- PDF 和 EPUB 各有阅读页骨架。
- 阅读页支持原文区和笔记区的基本布局。
- 笔记类型能根据颜色映射显示。
- 概念笔记能生成书内概念索引。
- 静态搜索索引能搜索书籍元数据、笔记和可抽取原文。
- `articles/` 可以保存微信公众号草稿，但不进入主站导航。
